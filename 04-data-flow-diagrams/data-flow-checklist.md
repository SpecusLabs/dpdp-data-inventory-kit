## How to use this checklist

Work through each section with your team. For each question, note:
- Yes / No / Partially
- The system or channel involved
- Any gaps or action items

Complete one pass for each category of Data Principal (customers, employees, candidates, etc.).

---

## Section 1 — Data Collection Points

These are the entry points where personal data first enters your organisation.

- [ ] We have identified every form on our website or app that collects personal data
- [ ] We have identified every offline channel through which personal data is collected (phone calls, paper forms, WhatsApp messages)
- [ ] We have identified every third-party source from which we receive personal data (data brokers, partners, purchased lists)
- [ ] Every collection point has a documented purpose
- [ ] Every collection point has a consent mechanism or a documented legitimate use basis
- [ ] Our consent notices accurately describe what data is collected at each entry point

## Section 2 — Data Storage Locations

These are all the places where personal data is stored after collection.

- [ ] We have a complete list of every system, database, and file store that contains personal data
- [ ] We have identified every spreadsheet or shared document that contains personal data
- [ ] We have identified every email inbox that regularly contains personal data
- [ ] We have identified every cloud storage location (Google Drive, Dropbox, OneDrive) that contains personal data
- [ ] We have identified every WhatsApp group or messaging channel that contains personal data
- [ ] Every storage location has an identified owner (person responsible)
- [ ] Every storage location has documented access controls
- [ ] Every storage location has a documented retention period

## Section 3 — Internal Data Movement

How data moves between systems within your organisation.

- [ ] We have mapped which systems automatically sync data between them (e.g. Shopify to Klaviyo via integration)
- [ ] We have mapped which data is copied manually between systems
- [ ] We have identified every automated workflow or API integration that transfers personal data
- [ ] We have identified every person who manually exports or transfers personal data as part of their role
- [ ] Every internal transfer has a documented purpose
- [ ] Access to move data between systems is restricted to authorised roles only

## Section 4 — Third-Party Data Sharing

How data exits your organisation to vendors, partners, or other third parties.

- [ ] We have identified every vendor who receives personal data from us
- [ ] We have a Data Processing Agreement (DPA) in place with every vendor who receives personal data
- [ ] We know exactly what personal data each vendor receives
- [ ] We know what each vendor does with the data we share
- [ ] We know whether each vendor uses sub-processors (their own vendors)
- [ ] We have confirmed that vendors delete data in accordance with our retention periods
- [ ] We have confirmed whether each vendor transfers data outside India

## Section 5 — Cross-Border Data Transfers

Data that leaves India.

- [ ] We have identified every instance where personal data is stored on servers outside India
- [ ] We have identified every vendor or sub-processor located outside India
- [ ] We have reviewed our cloud hosting configuration to confirm the data region (e.g. AWS Mumbai vs AWS Ireland)
- [ ] We are monitoring DPDP Rules guidance on cross-border transfers (rules not yet finalised as of 2025)
- [ ] We have disclosed international data transfers in our privacy policy

## Section 6 — Data Retention and Deletion

How personal data is removed when no longer needed.

- [ ] Every category of personal data has a documented retention period
- [ ] We have a mechanism to delete data from every storage system when the retention period expires
- [ ] We have a process to fulfil erasure requests under Section 12 of the DPDP Act
- [ ] We have identified which categories of data cannot be deleted due to legal or regulatory requirements
- [ ] We have a process to verify that data is deleted from vendor systems when we request deletion
- [ ] We have tested our deletion process (can we actually delete a specific person's data from all systems?)

## Section 7 — Informal and Uncontrolled Data Flows

Data that moves outside formal systems.

- [ ] We have identified personal data shared via personal WhatsApp (not WhatsApp Business)
- [ ] We have identified personal data shared via personal email accounts
- [ ] We have identified personal data in printed documents or physical files
- [ ] We have identified personal data shared verbally (e.g. read out over the phone) and whether records are kept
- [ ] We have policies in place to reduce or eliminate uncontrolled data flows
- [ ] Our team has been informed about acceptable and unacceptable ways to share personal data

## Section 8 — Children's Data

Data about individuals under 18 years.

- [ ] We have identified whether any of our systems collect data from individuals who may be under 18
- [ ] If yes: we have a verifiable parental consent mechanism in place (as required by Section 9 of the DPDP Act)
- [ ] We do not engage in behavioural tracking of individuals under 18 (prohibited under Section 9)
- [ ] Our privacy policy addresses how we handle data of minors

## DPDP Act references

| Section | Obligation |
|---|---|
| Section 4 | Personal data may only be processed for a lawful purpose |
| Section 6 | Consent must be free, specific, informed, and unambiguous |
| Section 8(4) | Reasonable security safeguards must be implemented |
| Section 8(7) | Data must be deleted when no longer necessary |
| Section 9 | Children's data requires verifiable parental consent |
| Section 11 | Data Principals have the right to access information about their data |
| Section 12 | Data Principals have the right to correction and erasure |
