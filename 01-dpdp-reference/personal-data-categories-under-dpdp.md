## Overview

The Digital Personal Data Protection Act, 2023 defines personal data broadly: any data about an individual who is identifiable by or in relation to such data (Section 2(n)).

This means the definition covers far more than obvious identifiers like names and phone numbers. It extends to indirect identifiers — data points that, when combined, can identify a person.

Use this reference when filling your data inventory to ensure you capture every category of personal data your business holds.

---

## Category 1 — Direct identifiers

Data that directly and unambiguously identifies an individual.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Full name | First name + last name | Almost all businesses |
| Mobile number | Indian mobile numbers, WhatsApp numbers | Almost all businesses |
| Email address | Personal and work email addresses | Almost all businesses |
| Government ID number | Aadhaar number, PAN number, Passport number, Voter ID | Staffing, fintech, KYC-heavy businesses |
| Date of birth | DD/MM/YYYY or year only | E-commerce, healthcare, fintech |
| Photograph | Profile photo, KYC selfie, ID scan | Staffing, fintech, apps with accounts |
| Signature | Wet or digital signature | Staffing, legal, financial services |
| Bank account number | Account number + IFSC | Payroll, fintech, B2C payments |
| UPI ID | UPI handle linked to a person | Payment processors, B2C apps |

**DPDP reference:** Section 2(n)

---

## Category 2 — Contact and location data

Data that can be used to reach or locate an individual.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Physical address | Home address, delivery address, office address | E-commerce, logistics, staffing |
| Pin code | Delivery pin code, home pin code | E-commerce, logistics |
| GPS coordinates | Real-time location, delivery location, check-in location | Logistics, field workforce apps |
| IP address | IPv4 or IPv6 address of user's device | SaaS, e-commerce, apps |
| City or region | City-level location derived from IP or registration | Analytics tools, SaaS |

**Note:** IP addresses are personal data when they can be linked to an identifiable individual, which is the case in most business contexts.

---

## Category 3 — Transactional data

Data generated through a person's transactions with your business.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Purchase history | Orders placed, items bought, amounts paid | E-commerce, retail |
| Payment status | Successful, failed, refunded transactions | E-commerce, fintech |
| Refund and return records | Reason for return, refund amount | E-commerce |
| Subscription status | Plan type, start date, renewal date | SaaS, D2C |
| Invoice and receipt data | GST invoice linked to a person | E-commerce, B2B SaaS |
| Cart and wishlist data | Abandoned cart items, saved products | E-commerce |

---

## Category 4 — Behavioural and digital data

Data generated by an individual's interactions with your digital products.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Device ID | Mobile IMEI, advertising ID, device fingerprint | Mobile apps, ad platforms |
| Cookie ID | First-party and third-party cookie identifiers | Websites, marketing platforms |
| Session data | Pages visited, time spent, click path | Analytics tools (GA4, Mixpanel) |
| Search history | Search queries within your platform | E-commerce, content platforms |
| App usage data | Features used, frequency of use, session length | SaaS, mobile apps |
| Advertising ID | GAID (Android) or IDFA (iOS) | Mobile apps, ad networks |
| Browser fingerprint | Browser type, OS, screen resolution combined | Ad tech, fraud prevention |

**DPDP reference:** Section 9 prohibits behavioural tracking of children. If any of this data may relate to users under 18, flag it.

## Category 5 — Communication data

Data contained in or derived from communications with your business.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Customer support records | Chat transcripts, email threads, call recordings | E-commerce, SaaS, staffing |
| WhatsApp messages | Messages sent via business WhatsApp | Staffing, D2C, sales teams |
| Form submissions | Contact forms, feedback forms, lead forms | Almost all businesses |
| Survey responses | NPS scores, feedback, ratings linked to a person | SaaS, e-commerce |
| Review content | Product reviews linked to a registered customer | E-commerce platforms |
| Complaint records | Formal complaints and resolution records | E-commerce, fintech |

## Category 6 — Employment and contractor data

Data held about employees, contractors, gig workers, and staffing candidates.

| Data Type | Examples | Commonly held by |
|---|---|---|
| Resume and CV | Work history, skills, education | Staffing, HR, any business that hires |
| Employment agreement | Signed offer letter, contract terms | All businesses with employees |
| Salary and payroll data | CTC, bank details, salary slips | All businesses with employees |
| Attendance records | Check-in/out times, leave records | Field workforce, manufacturing |
| Performance records | Appraisals, targets, feedback | All businesses with employees |
| Emergency contact details | Name and number of next of kin | Staffing, field workforce |
| Background verification data | BGV reports, reference check data | Staffing, regulated industries |
| PF and ESI data | Employee PF number, ESI registration | Businesses with formal employees |

---

## Category 7 — Sensitive personal data

The DPDP Act does not create a separate category of "sensitive" data in the same way as older frameworks, but certain data types warrant heightened caution.

| Data Type | Notes |
|---|---|
| Financial data | Bank account, card numbers, income details — high risk |
| Health and medical data | Diagnoses, prescriptions, disabilities — high risk |
| Biometric data | Fingerprints, face scans used for authentication — high risk |
| Religious or caste data | Identity data that can lead to discrimination |
| Sexual orientation or gender identity | Identity data that can lead to discrimination |
| Political opinions | Particularly relevant for platforms with social features |
| Children's data | Any data about individuals under 18 — requires parental consent (Section 9) |

---

## Category 8 — AI and model-specific data

Relevant for AI SaaS businesses processing personal data through machine learning models.

| Data Type | Examples |
|---|---|
| User-generated prompts | Text inputs to AI features that contain personal information |
| Model outputs | AI-generated content that reflects or reveals personal data |
| Training data | Datasets used to fine-tune models — may contain personal data |
| Inference logs | Records of model inputs and outputs stored for debugging |
| Embedding vectors | Numerical representations of text that may encode personal data |

## How to use this reference in your inventory

When filling any inventory template in this kit:

1. Work through this list category by category
2. For each category, identify whether your business holds any data of that type
3. For each data type you hold, identify which system(s) it lives in
4. Add a row to your inventory for each combination of data type + system

Do not try to do this from memory alone. Review your systems directly.
