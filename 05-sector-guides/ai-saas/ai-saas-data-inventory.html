DPDP Data Inventory — AI SaaS

Digital Personal Data Protection Act, 2023 | Specus Labs

⚠️ This is not legal advice. AI SaaS businesses have unique data flows — user prompts sent to third-party APIs, inference logs, and embeddings all require careful documentation. Consult a qualified legal professional.
AI SaaS-specific note: When users type into your product, their inputs may contain personal data. Those inputs are sent to a third-party AI API (OpenAI, Anthropic, Google, AWS Bedrock). The API provider is a Data Processor. A Data Processing Agreement is required with each provider. Inference logs storing those inputs are a personal data store — they need retention limits and deletion pipelines.

⚡ Priority Actions for AI SaaS Businesses

  • Sign DPAs with every AI API provider before handling real user data (OpenAI / Anthropic / Google / AWS)
  • Verify API inputs are NOT used for model training — opt out if default is opt-in
  • Set inference log retention to 30–90 days maximum and implement PII redaction
  • Include inference logs and embedding vectors in your account deletion pipeline
  • Audit training data for personal data before next fine-tuning run
  • Update privacy policy to disclose AI API providers, countries, and prompt data handling
